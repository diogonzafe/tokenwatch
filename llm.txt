# @diogonzafe/tokenwatch

## What this library does
Transparent TypeScript wrapper that intercepts LLM API calls (OpenAI, Anthropic, Google Gemini, DeepSeek) and tracks token usage and cost in real-time by session, user and model. Zero latency added. No changes required to existing code.

## Package
- npm: @diogonzafe/tokenwatch
- Node 20+, TypeScript 5.x, ESM + CJS dual build
- One runtime dependency: zod (config validation)
- Peer deps (all optional): openai, @anthropic-ai/sdk, @google/generative-ai, better-sqlite3

## Public API

### createTracker(config?) → Promise<Tracker>
Config fields (all optional):
- storage: 'memory' | 'sqlite'  — default 'memory'
- alertThreshold: number         — USD, fires webhook once when exceeded
- webhookUrl: string             — Discord/Slack webhook URL
- syncPrices: boolean            — fetch remote prices on startup (default true)
- customPrices: Record<string, { input: number, output: number }>

### Tracker methods
- track(entry)                   — called by providers, not by user
- getReport() → Report
- getModelInfo(model: string) → ModelPrice | null  — prices + maxInputTokens, null if unknown
- reset()
- resetSession(sessionId: string)
- exportJSON() → string
- exportCSV() → string

### Report shape
{
  totalCostUSD: number,
  totalTokens: { input: number, output: number },
  byModel: Record<string, { costUSD, calls, tokens: { input, output } }>,
  bySession: Record<string, { costUSD, calls }>,
  byUser: Record<string, { costUSD, calls }>,
  period: { from: string, to: string }
}

### Provider wrappers
- wrapOpenAI<T>(client: T, tracker): T
- wrapAnthropic<T>(client: T, tracker): T
- wrapGemini<T>(client: T, tracker): T
- wrapDeepSeek<T>(client: T, tracker): T  — alias for wrapOpenAI

All wrappers:
- Accept __sessionId and __userId as extra fields in the request params
- Strip those fields before forwarding to the real API
- Return the exact same type as the input client (generics preserve types)
- Track cost after a successful call; do nothing if the API throws
- Support streaming

## Price resolution order
1. customPrices (user override)
2. Remote prices.json from GitHub (cached 24h at ~/.tokenwatch/prices.json)
3. Bundled prices.json (always present as fallback)
Unknown model → cost = $0 + console.warn

Prices are in USD per 1 million tokens.
Cost formula: (inputTokens / 1_000_000) * price.input + (outputTokens / 1_000_000) * price.output

Model alias matching: exact match first, then prefix match (e.g. "gpt-4o-2024-11-20" matches key "gpt-4o").

## Bundled models (prices.json)
200+ models across OpenAI, Anthropic, Gemini, DeepSeek — auto-updated weekly.
Each model entry: { input: number, output: number, maxInputTokens?: number }
Source: https://github.com/BerriAI/litellm (model_prices_and_context_window.json)

## File structure
src/
  types/index.ts          — all exported types
  core/
    pricing.ts            — resolvePrice(model, layers), calculateCost(inputTokens, outputTokens, price)
    sync.ts               — getRemotePrices(), fetchRemotePrices(url?), loadCachedPrices()
    storage.ts            — IStorage, MemoryStorage, SqliteStorage, createStorage(type)
    tracker.ts            — createTracker(config) — main entrypoint
  providers/
    openai.ts             — wrapOpenAI
    anthropic.ts          — wrapAnthropic
    gemini.ts             — wrapGemini
    deepseek.ts           — re-exports wrapOpenAI as wrapDeepSeek
  index.ts                — public exports
bin/cli.ts                — CLI: sync | prices | report | help
prices.json               — bundled pricing data (200+ models, updated weekly)
scripts/scrape-prices.mjs — fetches LiteLLM JSON, run by GitHub Action
.github/workflows/sync-prices.yml — weekly price sync + npm publish

## CLI
tokenwatch sync     — fetch and cache remote prices
tokenwatch prices   — list bundled models and prices
tokenwatch report   — show last saved report (SQLite)
tokenwatch help     — help

## Config validation
zod schema validates TrackerConfig at createTracker() call time.
Throws "[tokenwatch] Invalid config:\n  <field>: <message>" on invalid input.

## SQLite
Optional. Requires better-sqlite3.
DB path: ~/.tokenwatch/usage.db
Falls back gracefully with clear error if better-sqlite3 not installed.

## Streaming
OpenAI: wraps AsyncIterable, captures usage from the last chunk (requires stream_options: { include_usage: true })
Anthropic: captures input_tokens from message_start event, output_tokens from message_delta event
Gemini generateContentStream: usage read from the response Promise after stream completes
