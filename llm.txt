# @diogonzafe/tokenwatch

## What this library does
Transparent TypeScript wrapper that intercepts LLM API calls (OpenAI, Anthropic, Google Gemini, DeepSeek) and tracks token usage and cost in real-time by session, user, model and feature. Non-blocking — track() is synchronous; report/reset/export methods are async. No changes required to existing code.

## Package
- npm: @diogonzafe/tokenwatch
- Node 20+, TypeScript 5.x, ESM + CJS dual build
- One runtime dependency: zod (config validation)
- Peer deps (all optional): openai, @anthropic-ai/sdk, @google/generative-ai, better-sqlite3, pg, mysql2, mongodb, @opentelemetry/api

## Public API

### createTracker(config?) → Tracker
Config fields (all optional):
- storage: 'memory' | 'sqlite' | IStorage  — default 'memory'; pass an IStorage instance for Postgres/MySQL/MongoDB
- alertThreshold: number                    — USD, fires webhook once when exceeded
- webhookUrl: string                        — Discord/Slack webhook URL
- syncPrices: boolean                       — fetch remote prices on startup (default true)
- warnIfStaleAfterHours: number             — warn if prices are older than N hours (0 = disable, default 72)
- customPrices: Record<string, { input: number, output: number, cachedInput?: number, cacheCreationInput?: number, maxInputTokens?: number }>
- budgets: {
    perUser?: BudgetConfig,       — alert when a single user's total spend crosses threshold
    perSession?: BudgetConfig,    — alert when a single session's total spend crosses threshold
  }
- suggestions: boolean            — log a hint after each call if a cheaper model in the same family has >50% savings (default false)
- anomalyDetection?: AnomalyDetectionConfig  — alert when a call is Nx above per-user/per-model rolling average
- exporter?: IExporter           — custom export hook called for every tracked entry (e.g. OTelExporter)

### AnomalyDetectionConfig
{ multiplierThreshold: number, webhookUrl: string, windowHours?: number, mode?: 'once' | 'always' }
- multiplierThreshold: fire alert when call cost > multiplierThreshold × rolling average (e.g. 3 = 3x above average)
- webhookUrl: Slack/Discord webhook to POST to
- windowHours: lookback window for baseline average (default 24)
- mode 'once' (default): fire once per entity (model or user) until reset(); mode 'always': fire on every anomalous call
- Checks two axes independently: per-user (if userId present) and per-model
- Minimum 1 prior call required — no alert on first call for a given entity
- Alert message format: "[tokenwatch] Anomaly: model "gpt-4o" call cost $X is Y.Zx above 24h average ($W)"

### IExporter interface
{ export(entry: UsageEntry): void | Promise<void> }
- Called synchronously after each storage.record() — fire-and-forget (errors never propagate to track() caller)
- Implement this interface to push entries to any custom sink (OTel, Datadog, custom DB, etc.)

### BudgetConfig
{ threshold: number, webhookUrl: string, mode?: 'once' | 'always' }
- mode 'once' (default): fires once per user/session; mode 'always': fires on every call that exceeds

### createLazyTracker() → LazyTracker
Returns a tracker that no-ops silently until init() is called. Safe for module-level singletons.
LazyTracker extends Tracker — structurally compatible everywhere a Tracker is expected.

LazyTracker extra method:
- init(config?: TrackerConfig): void — initializes with createTracker(config). Throws if called twice. Failed init() leaves tracker in no-op mode.

Before init() is called:
- track() → silent no-op
- getReport() → zero Report (totalCostUSD: 0, byModel: {}, etc.)
- getCostForecast() → zero CostForecast (burnRatePerHour: 0, basedOnPeriod: null)
- exportJSON() → '{}'
- exportCSV() → CSV header row only
- getModelInfo() → null
- reset() / resetSession() → Promise.resolve()

### Tracker methods
- track(entry)                              — called by providers, not by user (synchronous)
- getModelInfo(model: string) → ModelPrice | null  — prices + maxInputTokens, null if unknown (synchronous)
- getReport(options?: ReportOptions) → Promise<Report>
- getCostForecast(options?: ForecastOptions) → Promise<CostForecast>
- reset() → Promise<void>
- resetSession(sessionId: string) → Promise<void>
- exportJSON() → Promise<string>                      — full report as JSON string
- exportCSV() → Promise<string>                       — all raw UsageEntry rows as RFC 4180 CSV
  columns: timestamp,model,inputTokens,outputTokens,reasoningTokens,cachedTokens,cacheCreationTokens,costUSD,sessionId,userId,feature

### ReportOptions
{ since?: string, until?: string, last?: string }
- since/until: ISO 8601 date or datetime string
- last: shorthand string — '1h', '24h', '7d', '30d'
- last takes precedence over since when both are provided
- throws "Invalid \"last\" value" if last string is not recognised

### ForecastOptions
{ windowHours?: number }  — default 24; window for burn rate calculation

### Report shape
{
  totalCostUSD: number,
  totalTokens: { input: number, output: number },
  byModel: Record<string, { costUSD, calls, tokens: { input, output, reasoning, cached } }>,
  bySession: Record<string, { costUSD, calls }>,
  byUser: Record<string, { costUSD, calls }>,
  byFeature: Record<string, { costUSD, calls }>,
  period: { from: string, to: string },
  pricesUpdatedAt?: string   — date of the price data in use (e.g. "2026-04-22"), absent if prices not loaded
}

### CostForecast shape
{
  burnRatePerHour: number,             — USD per hour based on windowHours
  projectedDailyCostUSD: number,       — burnRatePerHour * 24
  projectedMonthlyCostUSD: number,     — burnRatePerHour * 24 * 30
  basedOnHours: number,                — actual span of entries used (0 if < 2 entries in window)
  basedOnPeriod: { from: string, to: string } | null   — null if basedOnHours is 0
}

### Provider wrappers
- wrapOpenAI<T>(client: T, tracker): WrappedOpenAI<T>      — augments create() params with TrackingMeta
- wrapAnthropic<T>(client: T, tracker): WrappedAnthropic<T> — augments create() params with TrackingMeta
- wrapGemini<T>(client: T, tracker): T                      — returns same type; meta on getGenerativeModel()
- wrapDeepSeek<T>(client: T, tracker): WrappedOpenAI<T>     — alias for wrapOpenAI

All wrappers:
- Accept __sessionId, __userId, and __feature as extra fields in the request params
- Strip those fields before forwarding to the real API
- wrapOpenAI covers: chat.completions.create (streaming + non-streaming) and embeddings.create
- wrapOpenAI reasoning tokens: reads completion_tokens_details.reasoning_tokens; priced as output tokens (separate from completion_tokens in o1/o3/o4)
- wrapOpenAI cached tokens: reads prompt_tokens_details.cached_tokens; priced at cachedInput rate (default 50% of input)
- wrapAnthropic reasoning tokens: approximate (thinking block chars ÷ 4); informational only — thinking is already included in outputTokens, not added to cost
- wrapAnthropic cached tokens: reads cache_read_input_tokens (priced at cachedInput rate) and cache_creation_input_tokens (priced at cacheCreationInput rate, default 125% of input)
- wrapGemini: __sessionId, __userId, __feature are passed on getGenerativeModel() params (not per generateContent call) — all calls from that model instance share the same meta
- Return the exact same type as the input client (generics preserve types)
- Track cost after a successful call; do nothing if the API throws
- Support streaming
- record() is fire-and-forget — a storage failure never interrupts the API call

## Prompt Caching Token Accounting

OpenAI caching: cached_tokens is a SUBSET of prompt_tokens.
- inputTokens stored = prompt_tokens - cached_tokens  (regular, non-cached)
- cachedTokens stored = cached_tokens
- totalTokens.input = inputTokens + cachedTokens (= prompt_tokens)
- Cost = regularInput * price.input + cached * price.cachedInput

Anthropic caching: input_tokens, cache_read_input_tokens, cache_creation_input_tokens are SEPARATE.
- inputTokens = input_tokens (regular)
- cachedTokens = cache_read_input_tokens
- cacheCreationTokens = cache_creation_input_tokens
- totalTokens.input = inputTokens + cachedTokens + cacheCreationTokens
- Cost = input * price.input + cached * price.cachedInput + creation * price.cacheCreationInput

## UsageEntry fields
model, inputTokens, outputTokens, reasoningTokens?, cachedTokens?, cacheCreationTokens?, costUSD, sessionId?, userId?, feature?, timestamp

## IStorage interface
All methods are sync-or-async (return T or Promise<T>):
- record(entry: UsageEntry): void | Promise<void>
- getAll(): UsageEntry[] | Promise<UsageEntry[]>
- clearAll(): void | Promise<void>
- clearSession(sessionId: string): void | Promise<void>

Built-in implementations: MemoryStorage (default), SqliteStorage (requires better-sqlite3)

## Database adapters (import from @diogonzafe/tokenwatch/adapters)

### PostgresStorage
- Peer dep: pg >= 8
- Constructor: new PostgresStorage(pool: pg.Pool | pg.Client)
- Extra method: migrate() → Promise<void>  — creates tokenwatch_usage table; safe to call multiple times (IF NOT EXISTS + incremental ALTER TABLE ADD COLUMN IF NOT EXISTS for databases created before v0.2.0/v0.3.0)
- Schema: id BIGSERIAL, model TEXT, input_tokens INT, output_tokens INT, reasoning_tokens INT NOT NULL DEFAULT 0, cached_tokens INT NOT NULL DEFAULT 0, cache_creation_tokens INT NOT NULL DEFAULT 0, cost_usd NUMERIC, session_id TEXT, user_id TEXT, feature TEXT, timestamp TIMESTAMPTZ

### MySQLStorage
- Peer dep: mysql2 >= 3
- Constructor: new MySQLStorage(pool: mysql2.Pool | mysql2.Connection)
- Extra method: migrate() → Promise<void>  — creates tokenwatch_usage table; safe to call multiple times
- Schema: id BIGINT UNSIGNED AUTO_INCREMENT, model VARCHAR(255), input_tokens INT, output_tokens INT, reasoning_tokens INT NOT NULL DEFAULT 0, cached_tokens INT NOT NULL DEFAULT 0, cache_creation_tokens INT NOT NULL DEFAULT 0, cost_usd DECIMAL(18,8), session_id VARCHAR(255), user_id VARCHAR(255), feature VARCHAR(255), timestamp DATETIME(3)

### MongoStorage
- Peer dep: mongodb >= 6
- Constructor: new MongoStorage(db: Db)
- Extra method: createIndexes() → Promise<void>  — creates indexes on timestamp, sessionId, userId, model
- Collection name: tokenwatch_usage
- Document fields: model, inputTokens, outputTokens, reasoningTokens? (omitted if 0), cachedTokens? (omitted if 0), cacheCreationTokens? (omitted if 0), costUSD, sessionId (null if absent), userId (null if absent), feature? (omitted if absent), timestamp

Custom adapter: any object implementing IStorage works as storage value.

## Price resolution order
1. customPrices (user override)
2. Remote prices.json from GitHub (cached 24h at ~/.tokenwatch/prices.json)
3. Bundled prices.json (always present as fallback)
Unknown model → cost = $0 + console.warn

Prices are in USD per 1 million tokens.
Cost formula (with caching):
  regularInputCost = (inputTokens / 1_000_000) * price.input
  cachedReadCost   = (cachedTokens / 1_000_000) * (price.cachedInput ?? price.input)
  cacheCreateCost  = (cacheCreationTokens / 1_000_000) * (price.cacheCreationInput ?? price.input * 1.25)
  outputCost       = (outputTokens / 1_000_000) * price.output
  total = regularInputCost + cachedReadCost + cacheCreateCost + outputCost

Model alias matching: exact match first, then prefix match (e.g. "gpt-4o-2024-11-20" matches key "gpt-4o").

## OpenTelemetry Exporter (import from @diogonzafe/tokenwatch/exporters)

### OTelExporter
- Peer dep: @opentelemetry/api >= 1.0.0 (optional; throws helpful error if not installed)
- Constructor: new OTelExporter(options?: OTelExporterOptions)
- OTelExporterOptions: { meterName?: string }  — default: 'tokenwatch'
- Requires a MeterProvider registered globally via the OTel SDK before instantiation
- Emits four instruments per tracked call:
    tokenwatch.calls         — Counter (+1)       attrs: model, session.id?, user.id?, feature?
    tokenwatch.input_tokens  — Counter (+inputTokens+cachedTokens+cacheCreationTokens)
    tokenwatch.output_tokens — Counter (+outputTokens)
    tokenwatch.cost_usd      — Histogram (costUSD, unit: USD)
- Usage: createTracker({ exporter: new OTelExporter() })

## CLI
tokenwatch sync                  — fetch and cache remote prices
tokenwatch prices                — list bundled models and prices
tokenwatch report [--db <url>]   — print cost breakdown (total, by model, by user, by session, by feature)
                                   default: ~/.tokenwatch/usage.db (SQLite)
                                   --db postgres://user:pass@host:5432/db
                                   --db mysql://user:pass@host:3306/db
                                   --db mongodb://user:pass@host:27017/db
tokenwatch dashboard [--port N]  — spin up local web dashboard at http://localhost:4242
             [--db <url>]        — same --db flag as report; works with any storage adapter
tokenwatch help                  — help

URL protocol auto-detection: postgres:// / postgresql:// → PostgresStorage (peer: pg)
                              mysql:// → MySQLStorage (peer: mysql2)
                              mongodb:// / mongodb+srv:// → MongoStorage (peer: mongodb)
                              (none) → SqliteStorage at ~/.tokenwatch/usage.db (peer: better-sqlite3)
Missing peer dep → clear error with install command, process exits cleanly.

## Dashboard
tokenwatch dashboard opens a dark-themed real-time web UI reading from ~/.tokenwatch/usage.db.
Sections: overview cards (total cost, tokens, calls, burn rate/hr), cost over time line chart, model doughnut chart + breakdown table, by-user and by-feature collapsible tables, cost forecast (projected daily/monthly).
Time filter tabs: 1h | 24h | 7d | 30d | All — switch without page reload via SSE reconnect.
SSE: server polls storage every 3s, sends data only when fingerprint changes (totalCostUSD + totalTokens.input + timeSeries.length).
Bucket intervals: 1h filter → 5-min buckets, 24h → 1h buckets, 7d/30d/all → 1-day buckets.
Zero new runtime dependencies: node:http only. Chart.js v4 from CDN (jsdelivr).
Port conflict: EADDRINUSE → helpful message + process.exit(1).

## Config validation
zod schema validates TrackerConfig at createTracker() call time.
Throws "[tokenwatch] Invalid config:\n  <field>: <message>" on invalid input.
Invalid storage string (e.g. 'redis') throws; valid IStorage instances pass validation via structural check.

## SQLite
Optional. Requires better-sqlite3.
DB path: ~/.tokenwatch/usage.db
Falls back gracefully with clear error if better-sqlite3 not installed.

## Streaming
OpenAI: wraps AsyncIterable, captures usage from the last chunk (requires stream_options: { include_usage: true })
  If stream has no usage data: records $0 + console.warn mentioning include_usage
Anthropic: captures input_tokens from message_start event, output_tokens from message_delta event; cache fields from message_start
Gemini generateContentStream: usage read from the response Promise after stream completes

## Agent Framework Integration

wrapOpenAI / wrapAnthropic do NOT work with agent frameworks — they never expose the raw SDK client. Use tracker.track() manually via each framework's usage callback.

tracker.track() always expects inputTokens and outputTokens. Field name mapping by framework:

Mastra (agent.generate onStepFinish):
  usage.promptTokens → inputTokens, usage.completionTokens → outputTokens
  model string format: "openai/gpt-4o" — extract model id manually

Vercel AI SDK v5 (streamText / generateText onFinish):
  usage.inputTokens → inputTokens, usage.outputTokens → outputTokens
  For multi-step agents use totalUsage (not usage) in onFinish for aggregate

LlamaIndex TypeScript (Settings.callbackManager.on('llm-end', handler)):
  event.detail.response.raw.usage.prompt_tokens → inputTokens (snake_case)
  event.detail.response.raw.usage.completion_tokens → outputTokens
  raw.model available for model name

LangChain.js — use built-in TokenwatchCallbackHandler (recommended):
  import { TokenwatchCallbackHandler } from '@diogonzafe/tokenwatch/langchain'
  new TokenwatchCallbackHandler(tracker, { defaultModel?, sessionId?, userId?, feature? })
  Handles tokenUsage (non-streaming) and estimatedTokenUsage (streaming) automatically.
  Extracts model from output.generations[0][0].message.response_metadata.model_name, falls back to defaultModel ?? 'unknown'.
  Peer dep: @langchain/core >= 0.1.0 (optional, not required at compile time).

  Manual alternative (BaseCallbackHandler.handleLLMEnd(output)):
    output.llmOutput?.tokenUsage.promptTokens → inputTokens
    output.llmOutput?.estimatedTokenUsage for streaming (approximate)
    model name not available in callback; set manually at handler init

## Model Suggestions
When suggestions: true in TrackerConfig, after each track() call:
- Detects provider family by model name prefix: gpt-, claude-, gemini-, deepseek-
- Compares cost against all models from the same family in the current price layers
- If cheapest alternative costs < 50% of current call: logs a console.log hint
- Merges layers: bundledPrices < remotePrices < customPrices (customPrices wins)
- No suggestion when: model unknown (costUSD=0), no cheaper alternative, savings ≤ 50%

## Privacy & Security
- Prompt and response content is never read or stored — only token counts and model names
- API keys are never accessed by tokenwatch
- All storage (SQLite, Postgres, MySQL, MongoDB) stays entirely in user's own infrastructure
- The Proxy wrapper makes no outbound network calls (only the price sync script fetches GitHub)
